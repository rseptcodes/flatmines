# 💣 FlatMines: Vanilla JS Minesweeper

A lightweight Minesweeper engine built from scratch with Vanilla JS and an optimized 1D flat array architecture.

This project is a functional engine for a classic Minesweeper game built entirely with Vanilla JavaScript. It serves as a practical exercise in state management, algorithm logic, and DOM manipulation without relying on modern frameworks.

## ✨ Features

* **1D Array Grid System:** The board state is managed using a single, flat 1D array rather than a nested 2D matrix. Spatial calculations (finding rows and columns) are resolved through standard math operations (`Math.floor` and modulo), keeping the data structure simple.
* **Recursive Flood Fill:** Uses a standard recursive function within `revealTile` to handle the cascade effect. When a player clicks a safe, empty tile (0), the game calculates and automatically reveals all adjacent safe tiles.
* **Dynamic CSS Integration:** The JavaScript code sets CSS custom properties (`--board-size`) dynamically. This allows the CSS Grid layout to adapt automatically to whatever board size is generated by the game logic.
* **Object-Based Structure:** The codebase is organized into distinct objects (`boardState`, `tilesManager`, `mainConfig`, `helperFunctions`) to separate data, UI management, and utility functions.
* **Bomb Distribution:** Automatically places bombs randomly based on a percentage of the total tiles, followed by a loop that increments the numerical values of all adjacent safe tiles.

## 🛠️ Technologies Used

* **Vanilla JavaScript (ES6+):** Core game logic, state arrays, and DOM element creation.
* **CSS3:** Uses CSS Variables and Flexbox/Grid for a responsive board, plus state classes (`.tiles--open`, `.tiles--bomb`) to handle visual updates.
* **HTML5:** Semantic structural shell where the JS injects the dynamic game elements.

## 🚀 Future Improvements

* **Event-Driven Architecture:** Implement the `eventBus` to fully decouple UI rendering from the core game logic, allowing state changes to trigger UI updates reactively.
* **File Separation (ES6 Modules):** Break down the single file into smaller, cohesive modules (`board.js`, `ui.js`, `events.js`) for better code maintainability.
* **Standard Mechanics:** 
  * Add Win/Loss conditions (triggering Game Over on bombs, winning when all safe tiles are found).
  * Implement right-click functionality to place flags on suspected bombs.
* **Difficulty Levels:** Add options for the user to choose different grid sizes and bomb amounts (e.g., Easy, Medium, Hard).
* **UI/UX Polish:** Improve the overall design, add a remaining bomb counter, a game timer, and prevent users from seeing the bombs in the DOM inspection before clicking.
